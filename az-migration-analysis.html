Known Limitations
What this analysis cannot tell you
🚫
No JavaScript-rendered content
The analysis fetches raw HTML. Client-side rendered components (React, Angular az-filter-items, Brightcove player) may not appear in static HTML. Investor-relations filter pages and any dynamic tabs were partially invisible.
🌏
Country hub geo-blocking
All 23 country hub pages fetched. 2 of 23 (Austria, global.html) returned empty HTML — likely geo-restricted. Their component signatures were not confirmed but the other 21 are consistent. Low risk: Austria/global would likely follow the same rich-header + icon-link-panel pattern.
🏠
Homepage partially fetchable
The homepage is heavily JS-rendered. Static HTML fetch confirmed homepage-hero + homepage-content-set + link-image-panel-v2 + image-panel-feature in the server-rendered markup, giving high confidence on block identity. Dynamic hero content and personalisation layers require a live browser audit.
📊
Press releases not uniformly validated
402 press releases sampled but the CSS signatures are so uniform (99% identical) that the rare exceptions (1% with hero-header, 2% with video) were treated as edge cases. Some legacy 2009–2012 pages may use rte-content instead of rich-text.
🔌
Third-party integrations invisible
Veeva ID footer, Adobe Analytics datalayer, OneTrust cookie banner, Brightcove video player, and reCAPTCHA are loaded dynamically. They appear as placeholders in static HTML. Each requires a dedicated integration story in EDS.
📐
Block ≠ authoring complexity
32 blocks is a component count, not a story-point count. A block appearing on 85% of pages (rich-text) may be trivial to build, while data-table appearing on 2% of pages may require a complex EDS spreadsheet integration. Effort sizing adjusts for this.

Key Risks & Decisions
⚠️
Press release archive scope — 1,606 pages back to 2009. Biggest single scope decision in the project.
🌏
Country sites localisation — Thailand's 173 pages need a dedicated locale strategy (separate site vs. multi-locale EDS).
🎥
Rich media / video — R&D pages contain many embedded videos. Need video hosting strategy (Brightcove? YouTube? Native EDS?).
💊
Veeva ID / MLR approval — Medical content carries Veeva IDs and approval dates. Content workflow must be validated with Regulatory.
🔗
Coupa supplier portal — AZ Suppliers section may require an external integration or iframe embed rather than native EDS.
📈
Pipeline / news feed blocks — Dynamic news feed on therapy area pages needs an EDS indexing-powered query.

Recommended Migration Phasing

Content-driven EDS development · ~28 weeks total
Phase 1
Foundation & Core Templates
Weeks 1–6 · ~50 story points
Global styles + fonts Header / Footer blocks hero block rich-text Press Release template Medical Release template Import script v1 (press releases) Navigation / breadcrumb
Start with the highest-volume, lowest-complexity content. Validates the EDS setup, content model, and import pipeline with real data early. Press releases alone represent 65% of the site — getting these right first delivers immediate value and de-risks volume.
Phase 2
Editorial & Thought Leadership
Weeks 7–12 · ~40 story points
Editorial Article template WSCD Topic template stats-callout block author-byline block related-articles block email-signup block references block topic-tags block
Medium-complexity editorial content. Articles and WSCD topics share many blocks — build them once, reuse across both templates. Veeva ID / MLR validation process should be confirmed with Regulatory team before this phase begins.
Phase 3
Deep Science & Therapy Content
Weeks 13–19 · ~45 story points
R&D Deep Content template Therapy Area Hub template jump-nav block video-embed block cards-grid block accordion block news-feed block (EDS indexing) pipeline table block
Highest per-page complexity. These pages require careful content modeling with authors before development. Video hosting strategy must be confirmed. The dynamic news-feed block requires EDS indexing to be set up and tested.
Phase 4
Corporate & Investor Content
Weeks 20–23 · ~25 story points
People Profile template Investor Relations template Sustainability pages timeline block download-list block publications-list block year-group block
People profiles are structured and consistent — good candidate for bulk migration with an import script. Investor relations annual report pages (going back to 2004) need a scope decision similar to press releases.
Phase 5
Homepage & Country Sites
Weeks 24–28 · ~40 story points
Homepage (bespoke design) hero-video block stat-counter block featured-articles block Country site setup (Thailand) Locale routing strategy Coupa/supplier integration Go-live performance audit
Homepage is deliberately last — it's unique and high-visibility, and benefits from having all block components already built. Country sites (especially Thailand) may spin off as a separate parallel stream once the core EDS setup is proven.
✓ Key Success Factors
• Content-first approach — model every template in Word/Google Docs before a single line of EDS code
• Import scripts for volume — press releases and people profiles should be migrated programmatically, not manually
• MLR/Veeva alignment — agree on how Veeva IDs and approval dates will be stored and displayed in EDS content model
• Lighthouse 100 baseline — establish performance baseline in Phase 1 and protect it throughout; easier to maintain than recover
• Archive decision early — press release scope (full archive vs. recent only) must be decided before Phase 1 ends
• Country site strategy — Thailand's depth warrants a dedicated decision on separate site vs. multi-locale before Phase 5
